# RUST 100% — Maximum Velocity Sprint Plan

**Дата:** 2026-05-31  
**Цель:** Сделать agentforge-runner полноценным Rust-оркестратором агентов как можно быстрее.

## Приоритетный порядок (что даёт максимальную скорость)

### Phase A — Foundation (самое важное прямо сейчас)

1. **JsonFileTaskStore + Atomic writes** (критично для персистентности)
2. **SQLite TaskStore** (для миграции с существующей tasks.db)
3. **Базовый CLI** (`agentforge task create/list/claim`, `run-worker`)
4. **Поддержка нескольких ключей** (MB2 + MB3) в конфиге и CLI

### Phase B — Usability & Power

5. Dynamic Router на Rust
6. Простой HTTP API (axum) — минимум 4 эндпоинта
7. `agentforge status` + мониторинг по ключам
8. Хорошая конфигурация (YAML)

### Phase C — Migration & Polish

9. Инструменты миграции из Python
10. Примеры и документация
11. Тесты
12. Удаление/деприкейшн Python оркестрации

## Конкретные задачи (готово к раздаче агентам)

Смотри задачи в базе с тегом `rust` + приоритет `critical`/`high`.

Рекомендация агентам:
- Бери задачи из Phase A первыми.
- Cloud Grok-4 (оба ключа) — бери сложные задачи (HTTP сервер, SQLite, CLI с ключами).
- Локальные агенты — гранулярные задачи + тесты + доки.

---

## Правила для максимальной скорости

- Все задачи должны быть маленькими и законченными (1-4 часа для сильного агента).
- Каждый PR/коммит должен быть reviewable.
- Используй существующий код в `agentforge-core` как основу.
- Если застрял — создавай подзадачу и двигайся дальше.

**Мы не ищем идеал. Мы ищем максимальную скорость движения к работающему Rust-оркестратору.**
