# AgentForge Repository Structure

**Last updated:** 2026-05-31 (during Code Management Professionalization initiative)

This document describes the layout of the AgentForge monorepo so that agents (Jules, Grok, etc.) and humans can navigate and contribute effectively.

## High-Level Layout

```
/home/agx/agentforge/
├── rust/                          # Rust workspace (core of the modern system)
│   ├── Cargo.toml                 # Workspace definition
│   ├── crates/
│   │   ├── agentforge-runner/     # Main binary (flywheel-step, continuous, candidate ops, rich export)
│   │   ├── agentforge-learning/   # TrajectoryDataset, SkillImprover, PRM, DPO/KTO/SFT
│   │   ├── agentforge-flywheel/   # Orchestration logic
│   │   ├── agentforge-candidates/ # Prioritization + promotion
│   │   ├── agentforge-planning/   # HierarchicalPlanner + LongTaskManager
│   │   ├── agentforge-safety/     # PolicyEngine
│   │   ├── agentforge-observability/
│   │   └── agentforge-core/       # Shared types (Outcome, Task, etc.)
│   └── target/                    # Build artifacts (gitignored)
│
├── learning/                      # Python learning layer (bridges + legacy)
├── eval/                          # Evaluation framework, trajectories, PRM, reports
├── bin/                           # Important operational scripts (make_*, enable/disable, audit, etc.)
├── agents/                        # Runners for different agents (jules_runner.sh, grok_runner.sh, ...)
├── services/ (systemd units)      # agentforge-*.service + timers
├── logs/                          # Runtime logs (gitignored)
├── pending_candidates/            # Generated by the flywheel (gitignored in normal use)
├── skills/                        # Playbooks / SKILL.md files
├── grok-work/                     # Work areas for agents (often worktrees or scratch)
│
├── *.sh, *.py (root)              # Core entrypoints and utilities
├── 100_PERCENT_READINESS_CHECKLIST.md
├── AGENTFORGE_FRONTIER_ROADMAP.md
├── HANDOFF_*.md
└── JULES_*.md                     # Integration docs for using Jules with the farm
```

## Key Concepts for Agents Working on This Repo

- **Pure Rust default** (as of 2026-05-31): Most orchestration now prefers `rust/target/release/agentforge-runner`.
- **Task-driven development**: Major work should be created as tasks in the internal task system (`http://localhost:8080/tasks`) with appropriate `preferred_agent` (jules / grok).
- **Trajectory capture**: Real work by agents should produce trajectories that feed the flywheel.
- **Dogfooding**: When improving AgentForge itself, we use the same agent conveyor we are building.

## Rust Workspace Notes

The Rust side is the strategic core. All new heavy lifting (flywheel, learning, planning, safety) should land here.

Build with:
```bash
cd rust
cargo build -p agentforge-runner --release
```

## How to Contribute (Agent Mode)

1. Create a task (or pick existing one with `preferred_agent` matching you).
2. Work in an isolated context (worktree recommended for local agents).
3. Produce clear trajectory / outcome.
4. The post-task hooks should automatically feed the Rust flywheel.
