# RUST 100% — Core Orchestration Goal

**Дата:** 2026-05-31  
**Решение:** Полный переход на 100% написанный на Rust код для оркестрации агентов и Flywheel.

## Стратегическая цель

Сделать **agentforge-runner + agentforge-core** полноценной заменой текущей Python-оркестрации (task_queue.py + mcp_server.py + workers).

Это значит:
- Rust Task Service становится каноническим.
- CLI (agentforge-runner) умеет запускать воркеров, создавать/распределять задачи, управлять flywheel.
- Минимальная зависимость от Python для core логики.

---

## Волна задач (Rust Core)

### Высокий приоритет (для Cloud Grok-4)

1. **Полноценная поддержка multiple keys в Rust Runner**
   - Добавить в agentforge-runner возможность работать с несколькими xAI ключами одновременно.
   - Балансировка нагрузки между ключами.
   - Конфигурация через файл или env.

2. **Rust Task Store с поддержкой приоритетов и preferred_agent**
   - Расширить InMemoryTaskStore + JsonFileTaskStore.
   - Добавить фильтрацию по preferred_agent, priority, complexity.
   - Реализовать claim / dispatch логику на Rust.

3. **CLI команда `agentforge run-worker`**
   - Сделать возможность запускать cloud и local workers прямо из agentforge-runner.
   - Поддержка `--key mb2`, `--key mb3`, `--both`.

4. **Базовый Dynamic Router на Rust**
   - Перенести логику выбора модели (по сложности задачи) из Python-воркеров в Rust.
   - Сделать её расширяемой.

5. **Интеграция с текущим task API (обратная совместимость)**
   - Сделать так, чтобы Rust runner мог работать поверх текущей SQLite БД задач (tasks.db), пока идёт миграция.

### Средний приоритет

6. **Мониторинг и статистика в Rust Runner**
   - Команда `agentforge status` — показывает запущенных агентов, нагрузку по ключам, расход.

7. **Поддержка Worktree в Rust** (для изоляции задач)
   - Аналог того, что сейчас делают grok_worker.sh через git worktree.

8. **Простой HTTP API на Rust** (замена части mcp_server)
   - Минимум: endpoints для создания задачи, получения списка pending, claim, update.

9. **Миграционный слой**
   - Скрипт/утилита, которая помогает постепенно переводить существующие Python-воркеры на вызовы agentforge-runner.

10. **Тесты и примеры использования**
    - Хорошие примеры запуска нескольких воркеров через Rust CLI.
    - Интеграционные тесты для TaskStore.

## Как будем работать

- **Cloud Grok-4 (оба ключа)**: Основная сила для написания нового Rust кода, архитектуры, сложного рефакторинга.
- **Local Grok**: Вспомогательные задачи, тесты, документация, мелкие правки.
- **Jules** (выборочно): Когда нужно написать большой чистый кусок кода и открыть PR.
